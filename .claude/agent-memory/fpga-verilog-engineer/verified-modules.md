Modules simulated X/Z-clean with passing assertions as of 2026-06-10:

- **uart_tx** (`fpga/src/uart_tx.v`, tb `fpga/sim/tb_uart_tx.v`): 8N1, idles high, param `CLKS_PER_BIT=234` for 27MHz/115200. Synchronous active-low reset. Verified 0x55 + 0xAA framing (LSB-first) and start-bit width = exactly 234 clocks.
- **adc_interface** (`fpga/src/adc_interface.v`, tb `fpga/sim/tb_adc_interface.v`): AD9226 12-bit capture, FPGA-generated ENCODE (adc_clk = 27MHz/8 = 3.375MHz). **Pipeline latency = 7 ENCODE cycles (AD9226 datasheet Rev B)**. **AD9226 outputs OFFSET BINARY — `sample_out <= {~adc_data[11], adc_data[10:0]}` converts to two's complement** (0x800->0x000, 0xFFF->0x7FF, 0x000->0x800). This is mandatory: cic_decimator declares `signed din`; feeding offset binary would put a 2048-LSB DC offset through DC-gain-512 and saturate the integrators. **`sample_otr` is a registered output**, latched on the same 7th ENCODE edge as `sample_out` (OTR is registered by the AD9226 with the same pipeline latency).
- **cic_decimator** (`fpga/src/cic_decimator.v`, tb `fpga/sim/tb_cic_decimator.v`): 3-stage R=8 CIC, 3.375MHz->421.875kSPS. **Output scaling is `c3 >>> 5` (NOT >>>12)** then signed-saturate to [-32768,+32767]. shift=5 = B_max-1-15 = 20-15; B_max=12+3*log2(8)=21. DC test: din 0x100 -> dout 0x1000. Integrators/combs MUST wrap (no saturation on them); only the output stage saturates. `decimate`/`dout_valid` are 1-cycle registered after the R-th `din_valid`.
- **fir_filter_bank1 / fir_filter_bank2** (`fpga/src/fir_filter_bankN.v`, tbs `fpga/sim/tb_fir_filter_bankN.v`): 32-tap symmetric Hamming-windowed bandpass, Q1.15 coeffs as compile-time constants. Bank1 center 36kHz, Bank2 center 44kHz, fs=421875 EXACTLY. Sequential 1-MAC-per-clock engine (IDLE/LOAD/DRAIN state machine, 1 HW multiplier each, ~N+3 cycles per sample, 64 sys-clks available). 38-bit accumulator, Q2.30 product >>>15 -> Q1.15, output saturates to signed 16-bit. dout_valid co-registered with dout, exactly 1:1 with din_valid. **dout is INTEGER sample scale, NOT Q1.15** — the >>>15 already removed the coeff scaling; downstream matched_filter reference chirp must use this integer scale (FIX-N2). FIR testbenches require iverilog `-g2012` flag (they use SystemVerilog local var declarations in unnamed blocks).
  - **FIX-W1 otr_in->otr_out: RESOLVED 2026-06-13 (testbench-only fix).** The DUT OTR logic was always correct (confirmed RESOLVED by dsp-signal-validator); the failure was a TB stimulus-timing race. **Pattern for testing a window-seeded OTR/flag input:** the DUT seeds its latch from otr_in in ST_IDLE on the SAME posedge it consumes din_valid (FIR), and the CIC commits otr_in on the R-th din_valid of the window. A 1-cycle otr_in pulse raised AFTER `wait(din_valid); @(posedge clk)` arrives too late / on a non-seeding edge and is cleared at the window boundary. **Fix: HOLD otr_in HIGH continuously across >1 full input window**, then inspect an output 2 windows later (`tgt = dout_count + 2`) — guarantees otr_in=1 on the seeding/committing edge. Second gotcha fixed at the same time: otr_out is a 1-cycle pulse co-registered with dout_valid, so sample it the instant `wait(dout_count >= tgt)` unblocks, BEFORE the next `@(posedge clk)` (otr_out has already dropped by then). Both FIR banks + CIC now ALL CHECKS PASSED.

**Why:** These are Week 3-5 deliverables on the FPGA critical path. The offset-binary MSB-flip, the >>>5 CIC shift, and registered sample_otr were systems-integrator consolidated fixes (FIX-B1, FIX-B2, FIX-W1) — earlier code passed adc_data raw, shifted >>>12 (42dB signal loss), and lacked the OTR port. SUPERSEDES the earlier (wrong) note that OTR was pass-through and sample_out was raw — that was reversed by hw-validation + dsp-signal-validator.
